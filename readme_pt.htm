Orologic: Relógio de Xadrez com PGN

Versão do Guia: 3.9.6

Orologic é uma aplicação para a gestão do tempo em partidas de xadrez, projetada para ser utilizada a partir da linha de comandos (consola). Ao contrário das aplicações com interface gráfica (GUI), Orologic não tem janelas, botões ou elementos gráficos interativos. A interação ocorre inteiramente através de texto: digitam-se comandos e leem-se as respostas fornecidas pelo programa.

Vantagens das aplicações de consola (CLI - Command Line Interface):

  • Acessibilidade: As aplicações de consola são intrinsecamente acessíveis a utilizadores com deficiência visual que utilizam leitores de ecrã (SR). Não havendo elementos gráficos, o leitor de ecrã pode ler diretamente o texto presente na consola, tornando a aplicação imediatamente utilizável.
  • Leveza: As aplicações de consola requerem menos recursos do sistema em comparação com as GUIs, tornando-as adequadas também para computadores menos potentes.
  • Automatização: As operações executadas a partir da linha de comandos podem ser facilmente automatizadas através de scripts.
  • Controlo preciso: A linha de comandos oferece um controlo muito preciso sobre as operações executadas.

Testes e Compatibilidade:

Orologic foi testado no Windows 11 com o leitor de ecrã NVDA versão 2024.3.1. No entanto, sendo uma aplicação Python, deverá funcionar corretamente também noutros sistemas operativos (ex. Linux, macOS) e com outros leitores de ecrã, desde que estejam instaladas as bibliotecas Python necessárias (ver abaixo).

Quem quiser utilizar os fontes pode cloná-los do github. No entanto, também terá de obter a biblioteca GBUtils, também no github.

Motor UCI:

Para as funcionalidades de análise, Orologic necessita de um motor de xadrez compatível com o protocolo UCI (Universal Chess Interface). Um motor UCI é um programa separado que analisa as posições de xadrez e sugere as melhores jogadas. Um exemplo muito popular e potente é o Stockfish.

Stockfish: Pode descarregar o executável do Stockfish (para vários sistemas operativos) a partir do site oficial: https://stockfishchess.org/download/

Depois de descarregar o Stockfish (ou outro motor UCI), terá de configurar o Orologic para indicar o caminho do executável (ver a secção "Configurar Motor").

Utilização

Para executar o Orologic, é necessário abrir um terminal (ou prompt de comando) na pasta onde se encontra o executável `orologic.exe` e digitar:

orologic.exe

e pressionar Enter.

Sugestões para a acessibilidade:

  • Recomenda-se aumentar a janela do terminal para facilitar a leitura por parte dos leitores de ecrã.
  • Além do prompt de comando clássico do Windows (cmd.exe), existe uma aplicação de terminal mais moderna e personalizável chamada Windows Terminal. Oferece funcionalidades como múltiplos separadores, painéis divididos, personalização de cores e tipos de letra, e uma melhor integração com os leitores de ecrã. Pode descarregar o Windows Terminal a partir da Microsoft Store: https://aka.ms/terminal

Ecrã Inicial

Ao iniciar, o Orologic mostra informações sobre a versão, data de criação e autor. Também fornece algumas estatísticas relativas à data de conceção e ao último lançamento. É também mostrada uma mensagem que convida a digitar `?` para visualizar o menu principal.

O menu principal é acessível digitando `?` e pressionando Enter. As opções disponíveis são:

Configurar Motor

Antes de poder utilizar as funcionalidades de análise, é necessário configurar o motor de xadrez. Selecione a opção motor no menu principal. O Orologic irá solicitar as seguintes informações:

  1. Caminho do executável do motor UCI: Insira o caminho completo do arquivo executável do motor (ex. C:\Engines\Stockfish\).
  2. Nome do executável do motor: Insira o nome do executável do motor, *com extensão* (ex. stockfish-windows-x86-64.exe).
  3. Tamanho da Hash Table (MB): O tamanho da memória (em megabytes) a alocar para a hash table do motor. Valores típicos são 128, 256, 512, 1024 (MB). Uma hash table maior pode melhorar o desempenho do motor, mas requer mais memória. O valor máximo é 4096.
  4. Número de núcleos a utilizar: O número de núcleos do processador que o motor pode utilizar. Um valor mais alto pode melhorar o desempenho, mas poderá tornar outras aplicações em execução mais lentas.
  5. Nível de habilidade (0-20): O nível de habilidade do motor (de 0, o mais fraco, a 20, o mais forte).
  6. Move Overhead (milissegundos): Um tempo adicional (em milissegundos) que o motor espera antes de retornar a jogada. Pode ser útil para simular um adversário humano ou para evitar uma utilização excessiva da CPU.

Estas definições são guardadas no arquivo orologic_db.json e carregadas automaticamente no próximo arranque.

Ajuste de Volume

A opção volume do menu principal permite ajustar o volume dos efeitos sonoros do Orologic. O valor vai de 0 (silencioso) a 100 (máximo). O volume predefinido é 50. O valor definido é guardado e restaurado automaticamente no próximo arranque.

Relógio

Criação de um Relógio

Quando seleciona "cria", a aplicação guia o utilizador através de uma série de perguntas para definir a configuração:

  1. Nome do relógio: Um nome único para identificar a configuração (ex. "Blitz 5+3").
  2. Brancas e Pretas começam com o mesmo tempo?:
    • Responda pressionando Enter (para *sim*) ou n (para *não*). Se sim, as definições de tempo serão iguais para ambos os jogadores.
  3. Fases: Um relógio pode ter até 4 fases. Para cada fase:
    • Tempo (hh:mm:ss): O tempo total disponível para a fase (ver Formato Horário).
    • Incremento em segundos: O incremento de tempo adicionado *após* cada jogada do jogador.
    • Número de jogadas: O número de jogadas a completar *dentro* da fase atual. Se definido como 0, a fase dura até ao fim da partida (útil para a última fase).
    • Se definir *Número de jogadas* como *0*, não será mais necessário responder às perguntas seguintes de configuração das fases.
  4. Alarmes: É possível definir até 5 alarmes sonoros:
    • Tempo (em segundos): O tempo restante em que o alarme soará. Por exemplo, um alarme aos 30 segundos soará quando o tempo restante do jogador ativo atingir os 30 segundos. Insere-se no formato *mm:ss*.
  5. Nota: Uma nota opcional para descrever a configuração (ex. "Regulamento do torneio").

Exemplo: Criação de um relógio Blitz 5+3

    Nome do relógio: Blitz 5+3
    Brancas e Pretas começam com o mesmo tempo?:  (Enter)
    Tempo (hh:mm:ss) para fase 1: 00:05:00
    Incremento em segundos para fase 1: 3
    Número de jogadas para fase 1: 0
    Número de alarmes a inserir (max 5, 0 para nenhum): 2
    Insira o tempo (mm:ss) para o alarme 1: 01:00
    Insira o tempo (mm:ss) para o alarme 2: 00:30
    Insira uma nota para o relógio (opcional):  (Enter)

Visualização dos Relógios Guardados

Esta opção (comando vê) mostra uma lista numerada dos relógios guardados, com as seguintes informações:

  • Nome do relógio.
  • Indicador B=P (se os tempos das Brancas e Pretas forem iguais) ou B/P (se forem diferentes).
  • Detalhes das fases (tempo e incremento para cada fase).
  • Número de alarmes.
  • Eventuais notas associadas.

Eliminação de um Relógio

Permite eliminar um relógio guardado (comando elimina). É visualizada a lista dos relógios (como em "Visualização"), e o utilizador pode selecionar o número correspondente ao relógio a eliminar.

Modificação das Informações PGN Predefinidas

Esta opção (comando pgn) permite modificar os valores predefinidos para as tags PGN que serão utilizadas nas novas partidas:

  • Event: Nome do evento (ex. "Campeonato Social"). O valor predefinido é "Orologic Game".
  • Site: Local do evento (ex. "Clube de Xadrez XYZ"). O valor predefinido é "Local desconhecido".
  • Round: Ronda de jogo (ex. "1"). O valor predefinido é "Round 1".

Estes valores predefinidos são utilizados quando se inicia uma nova partida, mas podem ser modificados também no início de cada partida.

Início de uma Partida

Para iniciar uma partida (comando joga):

  1. Seleção do relógio: É visualizada a lista dos relógios guardados. Selecione o relógio a utilizar inserindo o número correspondente.
  2. Inserção dos dados PGN:
    • Nome das brancas: Nome do jogador que conduz as peças brancas. Se deixado em branco, o valor padrão é "Brancas".
    • Nome das pretas: Nome do jogador que conduz as peças pretas. Se deixado em branco, o valor padrão é "Pretas".
    • Elo das brancas: Pontuação Elo do jogador branco. Se deixado em branco, o valor padrão é "?".
    • Elo das pretas: Pontuação Elo do jogador preto. Se deixado em branco, o valor padrão é "?".
    • Event, Site, Round: São solicitados estes valores, utilizando os valores predefinidos (ver Modificação das Informações PGN Predefinidas). É possível modificá-los ou aceitar os valores predefinidos pressionando Enter.
  3. Início da partida: Pressione Enter para iniciar a partida.

Comandos Durante a Partida

Durante a partida, o relógio do jogador ativo (inicialmente as Brancas) está em execução. O prompt mostra:

  • O número da jogada (ex. 1.).
  • A última jogada registada (se houver) (ex. e4).
  • Um prompt para inserir a próxima jogada (ex. 1. e4 ).
  • Se o jogo estiver em pausa, as secções acima indicadas serão mostradas entre parênteses retos (ex. [1. e4] ).

Estão disponíveis os seguintes comandos, que *começam com um ponto* (`.`):

Comando Descrição
.1 Mostra o tempo restante das Brancas.
.2 Mostra o tempo restante das Pretas.
.3 Mostra ambos os relógios.
.4 Compara os tempos restantes e indica qual jogador está em vantagem (e por quanto).
.5 Indica qual relógio está em andamento ou, se em pausa, a duração da mesma.
.l Visualiza a lista de jogadas realizadas.
.m Mostra o valor do material ainda em jogo.
.p Coloca em pausa/reinicia a contagem decrescente dos relógios. Quando em pausa, é visualizado o tempo decorrido em pausa.
.q *Apenas em pausa:* Anula a última jogada (tanto no tabuleiro como no PGN).
.b+X *Apenas em pausa:* Adiciona X segundos ao tempo das Brancas. X deve ser um número (ex. .b+5, .b+0.5).
.b-X *Apenas em pausa:* Subtrai X segundos ao tempo das Brancas.
.n+X *Apenas em pausa:* Adiciona X segundos ao tempo das Pretas.
.n-X *Apenas em pausa:* Subtrai X segundos ao tempo das Pretas.
.s Visualiza o tabuleiro atual.
.c comentário Adiciona um comentário à jogada *anterior*.
.1-0 Atribui vitória às brancas (1-0) e conclui a partida.
.0-1 Atribui vitória às pretas (0-1) e conclui a partida.
.1/2 Atribui empate (1/2-1/2) e conclui a partida.
.* Atribui um resultado indefinido (`*`) e conclui a partida.
.? Mostra a lista de comandos disponíveis.
/[coluna] Mostra a diagonal superior-direita a partir da base da coluna dada (ex. /a mostra a diagonal a1-h8).
\[coluna] Mostra a diagonal superior-esquerda a partir da base da coluna dada (ex. \h mostra a diagonal h1-a8).
-[coluna] Mostra as peças naquela coluna.
-[linha] Mostra as peças naquela linha.
-[casa] Mostra as peças ou os ataques naquela casa.
,[NomePeça] Mostra a(s) posição(ões) da peça indicada (ex. ,R mostra as torres brancas, ,n mostra os cavalos pretos).

Inserção das jogadas: As jogadas devem ser inseridas na notação algébrica SAN (Standard Algebraic Notation). Exemplos:

  • e4 (move o peão para e4)
  • Nf3 (move o cavalo para f3)
  • O-O (roque pequeno)
  • O-O-O (roque grande)
  • exd5 (o peão em e captura a peça em d5)
  • Bxf7+ (o bispo captura a peça em f7, dando xeque)
  • Qe8# (a dama move-se para e8, dando xeque-mate)
  • a8=Q (o peão em a7 move-se para a8 e promove a dama)
  • a8=Q+ (o peão em a7 move-se para a8 e promove a dama, dando xeque)

Após cada jogada inserida corretamente, a aplicação:

  1. Visualiza uma descrição verbal da jogada (ex. "o peão move-se para e4").
  2. Adiciona o eventual incremento ao tempo do jogador.
  3. Passa a vez ao outro jogador.

No final da partida (seja por abandono, xeque-mate, empate, tempo esgotado ou atribuição manual do resultado), o PGN completo da partida é guardado num arquivo. O nome do arquivo é composto por:

NomeBrancas-NomePretas-Resultado-DataHora.pgn

Exemplo: Carlsen-Anand-1-0-20240315143000.pgn

O Orologic perguntará se deseja analisar a partida recém-jogada.

Análise PGN

Introdução à Análise

O modo de análise (comando analisa) permite carregar e analisar partidas no formato PGN. É possível navegar entre as jogadas, adicionar comentários, avaliações, variantes e utilizar o motor de xadrez configurado para obter análises aprofundadas.

Ao entrar no modo de análise, o Orologic mostrará os cabeçalhos da partida e o número de jogadas. O PGN deverá estar na área de transferência. Caso não haja nenhum, o Orologic pedirá para copiar um PGN válido e depois pressionar l para carregá-lo.

Inspeção Inteligente das Linhas de Análise

Quando se visualizam as linhas de análise (comando e), é possível inspecioná-las no modo "inteligente". Este modo permite navegar dentro de uma linha de análise específica, jogada por jogada, visualizando a descrição verbal de cada jogada e a avaliação atualizada.

Depois de escolher a linha a analisar, os comandos disponíveis são:

Comando Descrição
s Vai para a jogada anterior.
d Vai para a jogada seguinte.
r Atualiza a avaliação da posição atual (recalcula com o motor).
? Visualiza a lista de comandos para o modo inteligente.
. Sai do modo de inspeção inteligente.

Formato Horário

O tempo, tanto para as fases do relógio como para os alarmes, deve ser inserido no formato hh:mm:ss:

  • hh: Horas (dois dígitos, ex. 01, 12).
  • mm: Minutos (dois dígitos, ex. 05, 59).
  • ss: Segundos (dois dígitos, ex. 00, 30).

Exemplos:

  • 01:30:00 (1 hora, 30 minutos)
  • 00:05:00 (5 minutos)
  • 00:00:10 (10 segundos)
